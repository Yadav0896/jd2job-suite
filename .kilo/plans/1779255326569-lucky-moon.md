# Meeting & Sales Copilot — Expansion Research & Plan

## 1. Your Idea Assessment

**Short answer: Yes, this is a very viable expansion.** The existing Interview Copilot codebase is already 60-70% of the way there. The core infrastructure — real-time audio capture, STT transcription, LLM-powered streaming suggestions — is generic enough to extend beyond interviews into general meetings and sales calls.

The market research confirms this is a **hot, competitive space** with clear gaps:

| What exists | What's missing (your opportunity) |
|---|---|
| Real-time transcription (Otter, Fireflies, Fathom, Tactiq) | Real-time *context-aware* coaching that feels intelligent, not keyword-spotting |
| Post-meeting summaries (everyone) | Suggestions/questions *during* the meeting, not just after |
| Sales coaching (Gong, Chorus — $120+/mo) | Affordable, bot-free, cross-platform alternative |
| Basic objection handling tips (Fireflies Live Assist, Otter) | Playbook-driven, deal-stage-aware, competitor-aware real-time suggestions |

Your unique angle vs the incumbents:
- **Bot-free capture** (desktop app) — users don't want a meeting bot visible to prospects
- **Real-time suggestions first** — most tools are post-meeting analysis with real-time as an afterthought
- **Context-aware** — resume + JD concept from interviews maps directly to **sales rep + lead/deal context**
- **Affordable** — the existing architecture costs pennies per call in API usage (Deepgram + NVIDIA)

## 3. Technical Expansion Roadmap

### Phase 1: Generalize the Core (Low effort, high impact)

Make the existing interview tool work for any meeting:

1. **Replace "Interview" terminology with "Meeting"** in the UI and state model
2. **Make the question detection pluggable** — different detectors for interview vs sales vs general
3. **Add meeting modes** — user selects: Interview | Sales Call | General Meeting | Standup
4. **Generalize the prompt system** — mode-specific system prompts (not hardcoded for interviews)
5. **Add a "Meeting Context" input** — like Resume+JD but for meetings: agenda, attendees, goals, deal stage

Files that change:
- `AppContext.jsx` — `interviewMode` → `meetingMode`, add `meetingContext` state
- `AudioControls.jsx` — make `isQuestion()` accept mode-specific detectors
- `nvidiaService.js` — mode-aware prompt builders
- `App.jsx` — add meeting type selector, remove interview-specific strings
- `TranscriptPanel.jsx` — speaker labels stay, question tag becomes "Action Item" / "Objection" / "Requirement"

### Phase 2: Post-Meeting Summary & Actions (Medium effort)

Replace `PostInterviewModal.jsx` with a proper meeting summary:

1. **Meeting Summary Generator**: Full summary with sections:
   - Overview (1-2 sentence summary)
   - Key discussion points (bullet list)
   - Decisions made
   - Open questions
   - Next steps / action items (with owners if detectable)
2. **Action Item Extraction**: Parse transcript with a specific LLM prompt to extract:
   - Tasks mentioned
   - Who owns each task
   - Deadlines mentioned
   - Priority indicators
3. **Export formats**: Plain text, Markdown, JSON (for CRM webhooks)
4. **Timeline view**: Visual timeline of the meeting with key moments marked

### Phase 3: Sales-Specific Features (Higher effort)

1. **Objection Detection**: Train/configure the question detector to recognize sales objections:
   - Pricing concerns ("too expensive", "budget")
   - Competition mentions ("we use X already")
   - Timing ("not now", "next quarter")
   - Authority ("need to check with")
2. **Real-time Playbook Suggestions**: When an objection is detected, surface the relevant response from a configurable playbook
3. **Buying Signal Detection**: Flag positive signals (agreement, budget mentions, timeline specifics)
4. **Competitive Intelligence**: If a competitor is mentioned, surface battle card snippets
5. **CRM Integration**: Webhook-based sync to Salesforce/HubSpot — push transcripts, summaries, action items

### Phase 4: Desktop App (Cross-platform)

The existing Electron scaffolding can be completed:
1. System tray icon for quick access
2. Global hotkey to toggle overlay (already partially works with Cmd+G)
3. No-bot audio capture — system audio + mic without joining as a meeting participant
4. Floating overlay widget that appears during meetings

## 5. Market Positioning Suggestion

Based on the research, the strongest positioning would be:

> **"Real-time meeting intelligence — without the bot."**

Key differentiators to emphasize:
1. **No meeting bot** — works via system audio capture, not as a visible participant
2. **Real-time first** — suggestions appear *during* the call, not after
3. **Works across platforms** — Zoom, Meet, Teams, Webex (since it captures from system audio)
4. **Two modes** — Interview Copilot (existing) + Sales Copilot (new) under one roof
5. **Affordable** — API-cost-based pricing vs $120+/seat for Gong

### Potential pricing:
- **Free tier**: Limited minutes/month, basic suggestions
- **Meeting Pro ($15/mo)**: Unlimited meetings, AI summaries, action items
- **Sales Pro ($29/mo)**: Objection handling, playbooks, CRM sync
- **Enterprise ($49/mo)**: Custom playbooks, team analytics, API access

Compare to: Gong ($120+), Chorus ($120+), Avoma ($19-39 + add-ons), Fireflies ($19/seat), Otter ($17-30)

## 6. Risks & Considerations

1. **System audio capture on macOS** — `getDisplayMedia` with `systemAudio: 'include'` is Chrome-based and requires user to check "Share Audio" in the dialog. This is a UX friction point. Consider a native desktop audio driver for seamless capture.
2. **Latency for real-time sales coaching** — objection → suggestion must be <1s. Current pipeline (STT + LLM) may need optimization. Consider streaming the LLM response and showing partial suggestions.
3. **Accuracy of action item extraction** — LLM-based extraction is imperfect. Consider using a dedicated model fine-tuned for this task, or allowing user verification.
4. **Competitor landscape is crowded** — differentiation via real-time + bot-free + affordable is strong but needs clear marketing.
5. **Privacy concerns** — recording meetings without consent is legally sensitive. Need clear consent flow, user awareness, and optional bot-based recording.
6. **NVIDIA API availability** — current reliance on a single API provider. Consider adding OpenAI/Azure/Anthropic as fallback options.

## 7. Recommendation

**Proceed with Phase 1 first.** It's the lowest effort and immediately validates the concept:

1. Rename/generalize the UI from "Interview" to "Meeting" (with Interview as one mode)
2. Add Meeting/Sales mode with different question detection and prompts
3. Replace the thank-you email modal with a proper meeting summary + action items
4. Ship this as v2.0

This gives you a working product in 1-2 weeks of development. Phases 2-4 can follow incrementally based on user feedback.

The existing codebase is a **strong foundation** — the audio pipeline, LLM streaming, and structured output display are all production-quality and generic enough to support this expansion.
